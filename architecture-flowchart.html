TestMind

How It Works — Business Overview

From a plain-English test description to a continuously running, self-healing automated test. No coding required — describe what your app should do, and the AI does the rest.

1

Author a Test

You set the objective — no coding required

📝
Describe your test in plain English
e.g. "Navigate to Onboarding, fill in Employee Name, submit the form, verify the confirmation banner appears"
🏷️
Select your Application Context
Tell the AI which app it's testing — SAP SuccessFactors, Workday, ServiceNow, or Custom — so it knows which UI patterns to expect
✨
Enrich Steps Optional
If you're not confident in the description quality, AI expands your steps into precise browser-level actions — explicit clicks, waits, and menu openings — before exploring
🔐
Set Auth Session If login needed
Name your saved login session — the AI uses it to skip the authentication wall and land directly inside the app
🚀
Trigger Exploration
Click Start — Studio dispatches the AI to run on a cloud-hosted GitHub Actions Runner. No infrastructure setup required on your side.
GitHub Actions Runner dispatched
2

AI Explores the App

AI drives a real browser, guided by your description

☁️
Fresh cloud VM spins up
GitHub Actions provisions a clean Ubuntu environment with a full Chromium browser and AI tooling installed — every run starts from a known clean state
🧠
Azure OpenAI plans the next browser action
GPT-4o reads your test description and reasons about what to do next — which button to click, what to type, where to navigate
🖱️
Playwright MCP drives the real browser
The AI executes clicks, form fills, and navigation on the live app. A screenshot is captured after every action as evidence of what happened.

4

Auto-Heal

AI fixes broken tests — you approve before anything changes

🔍
AI diagnoses the failure
AI reads the error message, the current Golden script, and the history of previous healing attempts — so it doesn't repeat a fix that already failed
🔧
Corrected Golden generated
AI produces an updated version of the test script. Every changed line is annotated with an explanation of why the change was made.
👁️
You review the diff in Studio
Studio shows the before-and-after side by side with the AI's reasoning. You decide whether the proposed fix is correct.
